¿De qué se trata?

El desarrollo web es una de las ramas más grandes y de mayor crecimiento en programación, debido a la accesibilidad desde cualquier lugar y dispositivo. Crear sistemas mediante tecnologías web es atractivo porque permite el acceso sin necesidad de instalaciones y con hardware/software básico. Este enfoque no solo busca crear sitios web, sino implementar sistemas, productos o aplicaciones utilizando tecnologías web.

Diferencias entre Diseño Web y Desarrollo Web

El diseño web se centra en la apariencia, la estética y la experiencia del usuario, abarcando aspectos como la navegación, colores y diseño gráfico. En cambio, el desarrollo web se ocupa de las funciones y características del sitio, como la programación, sistemas de registro, gestión de contenido y e-commerce, permitiendo que los usuarios interactúen con el sitio. Aunque ambos términos a veces se confunden, el diseño web se enfoca en la presentación, mientras que el desarrollo web aborda la funcionalidad.

¿De qué se trata la Arquitectura de Software?

La arquitectura de software es una guía estructural que define cómo se organiza, funciona e interactúan las partes de un sistema de software, similar a los planos de un edificio. A lo largo del tiempo, la programación ha evolucionado de ser un arte a seguir patrones y formas más organizadas para resolver problemas. Más allá de los algoritmos y estructuras de datos, la arquitectura se enfoca en el diseño global del sistema y su interacción, facilitando su desarrollo y mantenimiento.

Arquitectura Cliente-Servidor

La arquitectura Cliente-Servidor consiste en un cliente que envía peticiones y un servidor que las responde. Esta arquitectura es especialmente útil en sistemas distribuidos a través de redes. El servidor escucha en un puerto específico para recibir solicitudes, mientras que el cliente utiliza un puerto no reservado para enviar las peticiones. El cliente es un programa que establece conexiones activamente, con una vida útil finita, como un navegador web. El servidor es un programa que ofrece servicios de forma continua y puede manejar solicitudes concurrentes o secuenciales, con altos requerimientos de protección y autorización.

Características clave:
- Cliente interactúa con el usuario, servidor con los recursos compartidos.
- Diferentes requerimientos de recursos (memoria, procesador).
- Distinción clara entre roles de cliente (activo) y servidor (pasivo).
- Un servidor puede atender múltiples clientes.
- Es posible la conexión entre plataformas heterogéneas (diferente hardware y sistemas operativos).

Roles y Etapas en el Desarrollo Web

Desarrollador frontend
Programa la lógica del lado del cliente, maneja HTML, CSS, JavaScript y frameworks como React, Angular o Vue. Se encarga de la interacción con el usuario y la conexión con APIs.
Desarrollador backend:
Se ocupa de la lógica del lado del servidor, conectando con bases de datos y manejando frameworks como NodeJS, Django, Laravel, etc. Responsable de la lógica del negocio.
Diseñador UI/UX:
Se encarga del aspecto visual y la experiencia de usuario, trabajando con herramientas como Adobe XD, Figma o Sketch. No programa, sino que diseña la interfaz.
FullStack Developer:
Maneja tanto frontend como backend, y en algunos casos, bases de datos. Aunque cubre múltiples roles, no es responsable de todo el proceso, ya que no cubre áreas como el diseño.
Diseño:
Se establecen los flujos de usuario y se diseña la interfaz gráfica, definiendo la gama de colores y estilos, seguido del maquetado.

Desarrollo:
Se construyen las bases de datos, se programa la lógica del cliente y del servidor, y se dota de funcionalidad la interfaz. Es la etapa más larga y requiere coordinación.

Despliegue:
Tras completar el testeo, la aplicación se pone en producción y queda disponible para los usuarios.

Soporte:
Se corrigen errores, se implementan mejoras y se realiza el mantenimiento del sistema y base de datos.


Estas etapas están presentes en diversas metodologías de desarrollo de software.
¿Qué son HTML y CSS?
- (HyperText Markup Language) es el lenguaje básico utilizado para estructurar y representar el contenido de una página web, como texto e imágenes. Permite agregar "hipertexto", es decir, enlaces que conectan páginas web entre sí. Aunque define el contenido, no proporciona funcionalidad ni presentación.
- (Cascading Style Sheets) es el lenguaje utilizado para describir la apariencia visual de los documentos HTML. Define cómo los elementos deben mostrarse en la pantalla, incluyendo colores, fuentes y el diseño general. Mientras HTML estructura el contenido, CSS se encarga de su presentación visual.

¿Qué es un editor de texto?

Un editor de texto es un programa especializado para trabajar con archivos en texto plano, como los que se utilizan en programación. En este curso se recomienda utilizar Visual Studio Code, un editor popular y versátil.
Es fundamental tener un buen editor de texto para trabajar eficazmente durante el curso.

¿Cómo comienzo a usar CSS?

1. Crear archivo CSS:
- Abre Visual Studio Code y crea un nuevo archivo llamado **estilos.css** en el mismo directorio donde se encuentra tu archivo "index.html".
2. Escribir reglas CSS:
- En el archivo "styles.css" aplicamos los estilos deseados. Ejemplo: "css body { background-color: lightblue; } h1 { color: navy; } p { font-size: 18px; }"
3. **Vincular el archivo CSS al HTML**: - En el archivo **index.html**, agrega la siguiente línea dentro del ` ` para vincular el CSS: ```html ``` 4. **Guardar cambios**: - Asegúrate de guardar los cambios tanto en **index.html** como en **estilos.css**. 5. **Visualización**: - Abre el archivo **index.html** en tu navegador para ver los cambios visuales aplicados por el CSS. Con esto, has probado cómo incluir y aplicar CSS en tu página HTML.
